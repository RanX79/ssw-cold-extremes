# SSW Impacts Under Global Warming

Code and analysis workflow for investigating changes in surface-temperature
and stratospheric circulation impacts following sudden stratospheric warmings
(SSWs) under global warming using SEAS5 and ERA5 data.

## Repository structure

```text
.
├── scripts/
│   ├── 01_era5_ssw_dates.py
│   ├── 02_seas5_ssw_dates.py
│   ├── 03_figure1_seas5_ssw_t2m.py
│   ├── 04_figure2_seas5_ssw_t2m_lag.py
│   ├── 05_figure3_sliding_t2m_u.py
│   └── 06_figure4_cumulative_cold_anomaly_area.py
├── outputs/
├── README.md
├── LICENSE
└── requirements.txt
```

## Data availability

Raw ERA5 and SEAS5 data are not redistributed in this repository.
Users must obtain the data independently and update the input paths in
each script before running the analysis.

The scripts require:

- ERA5 daily zonal wind and temperature fields
- SEAS5 daily ensemble data
- SSW-event date tables generated by the included scripts

## Workflow

1. Run `01_era5_ssw_dates.py` to identify ERA5 SSW events.
2. Run `02_seas5_ssw_dates.py` to identify SEAS5 ensemble-member SSW events.
3. Run the figure scripts in numerical order.
4. Outputs and cache files are written to the configured `outputs/` directory.

## Python environment

The analysis was developed using Python 3 with:

- numpy
- pandas
- xarray
- scipy
- matplotlib
- netCDF4

Install dependencies with:

```bash
pip install -r requirements.txt
```

## Reproducibility notes

Before running a script, update its data paths, including the location of
the SEAS5 or ERA5 NetCDF files and the SSW-event CSV files. Intermediate
NPZ cache files may need to be deleted when settings such as the baseline
period, region definition, event window, or detrending method are changed.
